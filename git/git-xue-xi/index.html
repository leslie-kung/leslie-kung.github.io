选项说明

  • %H  提交对象（commit）的完整哈希字串
  • %h  提交对象的简短哈希字串
  • %T  树对象（tree）的完整哈希字串
  • %t  树对象的简短哈希字串
  • %P  父对象（parent）的完整哈希字串
  • %p  父对象的简短哈希字串
  • %an  作者（author）的名字
  • %ae  作者的电子邮件地址
  • %ad  作者修订日期（可以用 -date= 选项定制格式）
  • %ar  作者修订日期，按多久以前的方式显示
  • %cn  提交者(committer)的名字
  • %ce  提交者的电子邮件地址
  • %cd  提交日期
  • %cr  提交日期，按多久以前的方式显示
  • %s  提交说明

git commit操作之前文件的三种状态

  • Untracked files 未跟踪 需要 git add filename 来添加跟踪 (一般是新添加文件)
  • Changes not staged for commit 表明文件已经修改，但是还没有放入暂区域，也就是没生成快照 (一般是对已经跟踪的文件做了修改，可以用git checkout filename来放弃修改)
  • Changes to be committed 可以进行commit的文件
